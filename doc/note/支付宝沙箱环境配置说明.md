# 支付宝沙箱环境配置说明

## 一、获取支付宝沙箱环境配置信息

1. 访问支付宝开放平台沙箱环境：https://open.alipay.com/develop/sandbox/app
2. 登录支付宝开放平台账号（如果没有账号需要先注册）
3. 在沙箱环境中，你可以看到以下信息：
   - **APPID**：应用ID
   - **应用私钥**：需要自己生成（见下方说明）
   - **支付宝公钥**：沙箱环境提供的公钥

## 二、生成应用私钥和公钥

### 方式一：使用支付宝提供的工具生成（推荐）

1. 下载支付宝密钥生成工具：https://opendocs.alipay.com/common/02kipl
2. 运行工具，选择密钥类型为 **RSA2**（推荐）
3. 点击"生成密钥"，会生成：
   - **应用私钥**：用于签名
   - **应用公钥**：需要上传到支付宝开放平台

### 方式二：使用OpenSSL命令生成

```bash
# 生成私钥
openssl genrsa -out app_private_key.pem 2048

# 生成公钥
openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem

# 查看私钥内容（需要复制到配置中）
cat app_private_key.pem

# 查看公钥内容（需要上传到支付宝开放平台）
cat app_public_key.pem
```

**注意**：
- 私钥格式需要是 **PKCS8** 格式
- 如果生成的是PKCS1格式，需要转换：
  ```bash
  openssl pkcs8 -topk8 -inform PEM -in app_private_key.pem -outform PEM -nocrypt -out app_private_key_pkcs8.pem
  ```

## 三、上传应用公钥到支付宝开放平台

1. 在沙箱应用页面，找到"接口加签方式"
2. 点击"设置"，选择"公钥"
3. 将生成的**应用公钥**（不是私钥）粘贴进去
4. 保存后，支付宝会生成**支付宝公钥**，这个公钥需要配置到项目中

## 四、在Nacos中配置支付宝参数

在Nacos配置中心创建或更新配置文件：`novel-user-service-dev.yml`

```yaml
# 支付宝配置
alipay:
  # 应用ID（APPID）
  app-id: 你的APPID
  # 应用私钥（用于签名，需要是PKCS8格式，去掉头尾的-----BEGIN PRIVATE KEY-----和-----END PRIVATE KEY-----，只保留中间的内容，或者保留完整格式都可以）
  app-private-key: |
    -----BEGIN PRIVATE KEY-----
    你的应用私钥内容
    -----END PRIVATE KEY-----
  # 支付宝公钥（用于验签，需要是完整的公钥格式）
  alipay-public-key: |
    -----BEGIN PUBLIC KEY-----
    支付宝公钥内容
    -----END PUBLIC KEY-----
  # 网关地址（沙箱环境）
  gateway-url: https://openapi.alipaydev.com/gateway.do
  # 签名算法类型
  sign-type: RSA2
  # 字符编码
  charset: UTF-8
  # 返回格式
  format: JSON
  # 异步通知地址（支付成功后支付宝回调的地址）
  # 注意：需要是公网可访问的地址，本地开发可以使用内网穿透工具（如ngrok、natapp等）
  notify-url: http://你的域名或内网穿透地址/payment/alipay/notify
  # 同步跳转地址（支付完成后用户跳转的地址）
  return-url: http://你的域名或内网穿透地址/payment/alipay/return
```

## 五、本地开发环境配置

### 1. 使用内网穿透工具（推荐）

由于支付宝回调需要公网可访问的地址，本地开发需要使用内网穿透：

**使用ngrok：**
```bash
# 安装ngrok后，运行以下命令
ngrok http 9060
# 会生成一个公网地址，如：https://xxxxx.ngrok.io
# 将这个地址配置到notify-url和return-url中
```

**使用natapp：**
```bash
# 下载natapp客户端，运行
natapp -authtoken=你的token -subdomain=你的子域名 9060
```

### 2. 配置示例

假设使用ngrok，生成的地址是 `https://abc123.ngrok.io`，则配置如下：

```yaml
alipay:
  notify-url: https://abc123.ngrok.io/payment/alipay/notify
  return-url: https://abc123.ngrok.io/payment/alipay/return
```

## 六、测试支付流程

1. **创建充值订单**
   - 调用接口：`POST /payment/recharge`
   - 请求体：
     ```json
     {
       "rechargeAmount": 100,  // 充值金额（单位：分，100分=1元）
       "payChannel": 0  // 0-支付宝
     }
     ```
   - 返回结果中包含 `payUrl`，前端跳转到该URL进行支付

2. **使用沙箱账号支付**
   - 在支付宝沙箱环境中，有提供的测试账号和密码
   - 使用沙箱版支付宝APP扫码支付，或使用沙箱账号登录网页支付

3. **查看支付结果**
   - 支付成功后，支付宝会异步回调 `notify-url`
   - 同时用户会跳转到 `return-url`
   - 充值成功后，作者的付费积分会自动增加

## 七、常见问题

### 1. 验签失败
- 检查应用私钥和支付宝公钥是否正确配置
- 确认私钥格式是PKCS8格式
- 确认公钥已正确上传到支付宝开放平台

### 2. 回调地址无法访问
- 使用内网穿透工具，确保回调地址是公网可访问的
- 检查防火墙和网关配置，确保回调接口可以正常访问

### 3. 订单状态未更新
- 检查异步通知接口是否正常处理
- 查看日志，确认是否有异常信息
- 确认数据库连接正常

### 4. 积分未增加
- 检查Redis连接是否正常
- 查看日志，确认积分增加逻辑是否执行
- 检查作者ID是否正确

## 八、生产环境配置

生产环境配置与沙箱环境类似，主要区别：

1. **网关地址**：使用正式环境地址
   ```yaml
   gateway-url: https://openapi.alipay.com/gateway.do
   ```

2. **APPID和密钥**：使用正式环境的APPID和密钥

3. **回调地址**：使用生产环境的域名

4. **应用审核**：需要在支付宝开放平台提交应用审核，审核通过后才能使用

## 九、相关接口说明

### 创建充值订单
- **接口**：`POST /payment/recharge`
- **需要登录**：是
- **需要作者身份**：是
- **请求体**：
  ```json
  {
    "rechargeAmount": 100,  // 充值金额（分）
    "payChannel": 0  // 0-支付宝
  }
  ```
- **响应**：
  ```json
  {
    "code": 0,
    "msg": "success",
    "data": {
      "outTradeNo": "RECHARGE1234567890",
      "payUrl": "https://openapi.alipaydev.com/gateway.do?..."
    }
  }
  ```

### 支付宝异步通知（内部接口）
- **接口**：`POST /payment/alipay/notify`
- **说明**：支付宝回调接口，不需要认证

### 支付宝同步返回（内部接口）
- **接口**：`GET /payment/alipay/return`
- **说明**：支付完成后用户跳转的页面

## 十、积分兑换规则

- **1元 = 100积分**
- 充值金额单位：分（100分 = 1元）
- 充值后获得的积分会增加到作者的**付费积分**中
- 付费积分可以用于AI服务消费，且不会过期
