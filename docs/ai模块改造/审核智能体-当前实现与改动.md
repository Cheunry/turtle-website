## 二、当前审核总览

### 1. 调用路径

```
MQ（BookAuditRequestListener / ChapterAuditRequestListener）
  或 InnerAiController（/inner/.../audit/book|chapter）
       → TextServiceImpl.auditBook | auditChapter
       → AuditPipeline.execute(context)
       → 各 AuditStep（责任链）
```

### 2. 书籍审核流水线（`BookAuditPipelineFactory`）

| 顺序 | Step | 作用 |
|------|------|------|
| 1 | `BookValidateStep` | 参数校验 |
| 2 | `BookSensitiveWordFilterStep` | 本地敏感词；**学习资料类（见配置 `learning-audit.category-id`，默认 8）跳过** |
| 3 | `BookPromptAssembleStep` | 渲染 system/user；小说类拼 Nacos `audit-category`；**学习资料用 `book-audit-learning`，不拼 audit-category** |
| 4 | `BookLlmInvokeStep` | `StructuredOutputInvoker` + `ChatClient`；**学习资料不挂载 RAG Advisor** |
| 5 | `BookResponseBuildStep` | 组装 `BookAuditRespDto` |

### 3. 章节审核流水线（`ChapterAuditPipelineFactory`）

| 顺序 | Step | 作用 |
|------|------|------|
| 1 | `ChapterValidateStep` | 参数校验 |
| 2 | `ChapterSensitiveWordFilterStep` | 本地敏感词；**学习资料类跳过** |
| 3 | `ChapterSegmentStep` | 按字数切分；小说默认 `novel.ai.chapter.max-content-length`（默认 5000）；学习资料用 `learning-audit.segment-chars` / 绿色通道 `green-channel-segment-chars`；可选绿色通道截断正文前 N 字并写入 `ChapterAuditContext.learningAuditNote` |
| 4 | `ChapterSegmentAuditStep` | 逐段 LLM；小说类挂 RAG（若全局启用且 Bean 存在）；**学习资料用 `chapter-audit-learning` 且不挂 RAG** |
| 5 | `ChapterResponseBuildStep` | 合并多段结果 |

### 4. ChatClient 分工（与 tool-and-mcp 一致）

- **`chatClient`（`AiConfig`）**：默认 Advisors + **`AuditTools`**，用于**需要工具**的链路（审核 LLM 仍可能触发工具调用）。
- **`textChatClient`**：无 Tools，用于润色、封面、审核规则抽取等，避免无关场景携带大工具定义。

学习资料路径虽**不挂 RAG**，但仍使用带 `AuditTools` 的 `chatClient`；若需彻底禁止工具侧写向量等，需后续单独拆「无工具 ChatClient」分支。

### 5. 观测与异常

- Pipeline 监听器含 `LoggingPipelineListener`、`SkywalkingPipelineListener`、`MicrometerPipelineListener`（见各 `*AuditPipelineFactory`）。
- 结构化调用日志见 `StructuredOutputLogAdvisor`；瞬时错误重试见 `RetryTransientAiAdvisor`。

## 三、按书籍类别（`category_id`）的策略

- **请求字段**：`BookAuditReqDto` / `ChapterAuditReqDto`（及 MQ DTO）含 `categoryId`、`categoryName`、`authorId`。
- **业务侧**：`BookSubmitListener`、`ChapterSubmitListener` 从 `BookInfo` 填充；书名/简介变更送审时从库中 `fresh` 记录带类别与作者。
- **配置**：`NovelAiAuditCategoryProperties`，前缀 **`novel.ai.audit-category`**：`category-guidelines`（key 为类别 ID 字符串）、`default-guidelines`。
- **解析**：`AuditCategoryPromptResolver` 将命中文案追加到 **非学习资料** 的 `book-audit` / `chapter-audit` system 侧。
- **说明**：未在审核链路使用 `workDirection`（男频/女频）。

---

## 四、学习资料类（默认 `categoryId = 8`）

- **配置**：`NovelAiLearningAuditProperties`，前缀 **`novel.ai.learning-audit`**：`category-id`、`segment-chars`、`green-channel-segment-chars`、`green-channel-max-audit-chars`、`green-channel-author-ids`。
- **跳过本地敏感词**：避免小说词库误伤代码/术语等。
- **不挂载 RAG**：不做判例向量检索。
- **独立模板**：`BOOK_AUDIT_LEARNING`、`CHAPTER_AUDIT_LEARNING`（`resources/prompts/book-audit-learning-*.st`、`chapter-audit-learning-*.st`）。
- **通用模板**：`book-audit-system.st` / `chapter-audit-system.st` 含「体裁认知」，避免用小说标准卡教辅体例。

## 五、Nacos / 配置键一览

| 前缀 | 用途 |
|------|------|
| `novel.ai.audit-category` | 非学习资料：按类别附加审核说明 |
| `novel.ai.learning-audit` | 学习资料：类别 ID、分段、绿色通道作者、可选截断 |
| `novel.ai.chapter.max-content-length` | 小说类章节分段默认长度 |
| `novel.ai.rag.enabled` | 是否装配 RAG 相关 Bean；学习资料路径仍**不挂载** Advisor |
| `novel.ai.sensitive-filter.*` | 本地敏感词词典 |
| `novel.ai.policy.*` | 政策工具（见 `novel-ai-policies.yml` 与 tool-and-mcp） |

具体示例见仓库 `nacos/local/DEFAULT_GROUP/novel-ai-service-dev.yml`、`nacos/prod/DEFAULT_GROUP/novel-ai-service-prod.yml`。
