# 常见问题与排查（初稿）

本页优先沉淀“线上最常见、面试最常被问”的问题。

## 1. 网关启动正常，但请求 5xx

**排查顺序**

1. 看网关日志，确认路由是否命中。  
2. 看下游服务是否健康、是否注册到 Nacos。  
3. 看 Nacos 配置是否缺失或误配。  
4. 区分是网关报错，还是下游服务原样透传的 5xx。  

**重点命令**

```bash
docker compose -f /home/ubuntu/project/docker-compose.prod.yml logs -f novel-gateway
```

## 2. Nacos 连接异常（UNHEALTHY）

**现象**
- 日志出现 `Client not connected, current status:UNHEALTHY`

**定位**
- 检查 Nacos 地址端口可达性
- 检查认证信息（如启用用户名密码）
- 检查服务端防火墙与安全组

**参考**
- `doc/note/NACOS_TROUBLESHOOTING.md`

## 3. 浏览器出现跨域错误（CORS）

**常见现象**
- 控制台报 `CORS policy` 或 `No 'Access-Control-Allow-Origin' header`

**排查步骤**

1. 确认请求是否经过网关（而非直接访问后端服务）。  
2. 检查网关 `CorsWebFilter` 是否生效。  
3. 检查前端请求是否携带凭证（Cookie）且与跨域配置匹配。  
4. 检查预检请求（OPTIONS）是否被正确响应。  

**说明**
- 当前网关使用 `allowedOriginPattern("*")` + `allowCredentials(true)`，联调方便。
- 生产环境建议收敛为明确域名白名单，降低安全风险。

## 4. 搜索不到新上架书籍

**常见原因**
- 书籍变更消息未被搜索服务消费
- ES 写入失败
- 书籍审核状态未满足可检索条件

**排查**

1. 看 `novel-book-service` 是否成功发送消息。  
2. 看 `novel-search-service` 消费与入 ES 日志。  
3. 直接在 ES 检查对应文档是否存在。  

## 5. 作者侧接口响应慢

**常见原因**
- 同步调用链过长
- 缓存命中率低
- MQ 发送或下游消费异常导致重试

**排查**

1. SkyWalking 查看慢点位于网关、用户服务还是下游。  
2. 检查 Redis 命中情况。  
3. 检查 RocketMQ 发送耗时与积压。  

**优化参考**
- `doc/note/性能优化总结文档.md`

## 6. 网关 504 超时

**常见原因**
- 下游服务耗时过长（如 AI 相关请求）。
- 网关超时配置小于业务实际耗时。

**排查步骤**

1. 在链路追踪中确认慢点位于网关还是下游。  
2. 查看网关与下游服务的超时参数是否匹配。  
3. 检查下游线程池、外部依赖（AI API、数据库、ES）是否瓶颈。  

## 7. 部署后服务起不来

**排查顺序**

1. `docker ps -a` 看容器是否反复重启。  
2. 看服务日志中的首个异常栈。  
3. 检查配置中心参数与环境变量。  
4. 检查依赖中间件连通性（Nacos/Redis/MQ/ES/MySQL）。  

## 8. 常用排查原则（建议固定执行）

- 先看入口（网关）再看下游，避免盲查。  
- 先确认配置与依赖可用，再看业务代码。  
- 先定位“第一处报错”，不要被连锁异常干扰。  
- 故障闭环后补到本页：现象、根因、修复、预防。
