## **项目**

### [三高电子书站-网站链接：http://120.53.89.103:1024](http://120.53.89.103:1024)

`Spring Cloud Alibaba` `MySQL` `Redis`  `Elasticsearch` `RocketMQ` `Spring AI`  `XXL-JOB` `SkyWalking` `Docker` `RAG`

- **项目描述**：
分布式在线小说阅读平台，包含门户、作家后台、管理后台等子系统。基于 Spring Cloud Alibaba 微服务架构，拆分为用户、书籍、搜索、网关、AI 服务模块。
- **工作内容**

1. 基于 Spring Cloud Alibaba 微服务架构，独立完成用户、书籍、搜索等核心模块的设计开发，实现全链路容器化部署与编排。
2. 针对搜情节等复杂查询语义理解弱的痛点，基于 Elasticsearch 设计 KNN 向量检索 + BM25 文本检索的并行召回架构，应用层引入 RRF 算法重排序，提升复杂查询准确率。
3. 基于 RocketMQ 事务消息重构书籍发布链路，实现数据库与 ES 索引构建的异步解耦，核心接口响应时间从 274ms 降至 76ms（降低 72%）。
4. 设计 Redis + Caffeine 多级缓存体系配合 XXL-JOB 实现 Write-Behind 策略，将高频写操作转化为批量异步入库；通过 Pipeline + Lua 脚本优化 Redis 交互，网络 IO 减93%，榜单查询性能提升 90%+。
5. 根据压测慢查询日志 + EXPLAIN 分析，重构核心业务 SQL 与复合索引，同步优化 HikariCP 连接池配置，核心接口 QPS 从 60 提升至 200。
6. **审核链路**：基于 **Spring AI** 将书籍/章节审核编排为 `AuditPipeline` 多 Step 责任链（校验、敏感词、模板、LLM、结构化出参），支持**短路**终止后续步骤。**敏感词**使用 **Aho-Corasick 双数组 Trie**（`AhoCorasickDoubleArrayTrie`）做多模式匹配，对正文近似 **O(n)** 单次扫描；命中则直接拒审，避免再调向量检索与大模型。**章节**按配置**字数切段**，控制单次请求的上下文长度。
7. **Advisor 与 Token 观测**：`ChatClient` 挂载 **Advisor 链**：外层 `RetryTransientAiAdvisor`** 仅对**瞬时错误**（限流、网关 5xx 等）指数退避，**内容安全等非瞬时错误不重试**；内层 `StructuredOutputLogAdvisor` 从模型返回的 `Usage` 记录 **prompt / completion / total Token** 日志，并可打 **SkyWalking** `ai.call.token.`* 标签，与 `gen_ai.*` Observation** 互补。**润色、封面**等使用**无 Tools** 的独立 `textChatClient`，避免工具定义进入 Prompt，**省 Token、降延迟**。
8. **分类审核与 RAG**：按 `categoryId` 从 **Nacos** 注入品类审核说明；**学习资料**独立模板、**不挂 RAG**、可跳过小说敏感词并对绿色通道**截断正文**控成本。判例 **RAG** 为**自建 ES 向量索引**，**仅在**书籍/章节 **LLM 步骤** 挂载检索增强。全系 **SkyWalking**（链路存 **ES**）；流水线 **Micrometer**（`novel.ai.audit.`*）统计*耗时与决策*，与 **Token 粒度**埋点分工。相较改造前，**违规内容识别准确率提升约 25%，人工复审率降低约 40%**。
9. **基于 Spring AI 构建多阶段 Agent 审核流水线，设计基于分类索引的动态策略路由与 Advisor 链治理体系，集成“人工纠偏”驱动的判例 RAG 增强与 AC 自动机过滤，实现审核准则的秒级热更新，违规识别准确率提升 25%。**
